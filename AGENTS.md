## Core Behavioral Principles

**Think before coding.** State assumptions explicitly. Surface tradeoffs. If uncertain, ask.

**Simplicity first.** Minimum code, no speculative abstractions, no unrequested features. If 200 lines could be 50, rewrite.

**Surgical changes.** Match existing style. Don't refactor unrelated code. Every changed line must trace to the request.

**Goal-driven execution.** Transform tasks into verifiable goals. Loop until verified. Multi-step tasks get a plan with checkpoints.

## Code Style

### Imports
```nim
import std/[options, strformat]   # stdlib first
import ./[ffi, gtypes, error]     # local modules after
```

### Naming
- Types: **PascalCase** — `Array[T]`, `ArrayBuilder[T]`
- Procs, funcs, iterators, templates, vars, fields: **camelCase** — `newArrayBuilder`, `appendNull`
- Enums: non-pure prefix values (`pcFile`), pure use PascalCase
- Generic params: `T` for single, descriptive for multiple
- C FFI types: keep `GArrow` prefix — `GArrowArray`
- Test files: `test_*.nim`

### Proc vs Func vs Template vs Macro
- Default to **`proc`**
- Use **`func`** for pure helpers/accessors when checked purity helps
- Use **`template`** only for tiny substitutions or syntax wrappers
- Use **`macro`** only when syntax transformation is required
- Use **`method`** only when runtime dispatch is needed
- Prefer top-level helpers. Use nested procs only for truly local logic or intentional closures.
- A nested proc captures outer locals by default; mark `{.nimcall.}` to keep it non-capturing.

### Calls, Locals, Types
- Use `let` by default; `var` only for mutating values
- Keep local declarations close to first use
- Keep `type` blocks at module scope
- Prefer compact wrapped calls over one-argument-per-line
- Use UFCS when it reads like an accessor
- Use `func` for pure accessors
- For object constructors, override only the fields you change

### Formatting
- 2 spaces, no tabs
- 80 char soft limit; wrap before lines become hard to scan
- No column alignment with extra spaces
- Slice notation: `a..b` unless spaces needed for clarity
- Format with `nimble format` (nph)

### Whitespace Sensitivity (compile errors if violated)
- **No space before `[`** in type annotations: `array[0..4, bool]` ✓, `array [0..4, bool]` ✗
- **No space between proc name and `(`** with 2+ args: `foo(1, 2)` passes two ints; `foo (1, 2)` passes a tuple
- **Spaces required around `..` and `..<`**: `0 .. high(s)`, `0 ..< s.len`

### Line Continuation
A statement continues when the current line ends in: an operator (`and`, `or`, `+`), a comma `,`, or an opening bracket `(` `[` `{`. The continued line must be indented further.

### Control Flow
- Prefer straightforward `if/elif/else` and explicit loop conditions
- Use early `return` for guard exits or found values, not as the default shape
- Do not use `continue` — restructure the branch
- In multi-step procs, keep one clear normal path

## Error Handling

### The `check` Macro (in `src/narrow/core/error.nim`)
Wraps C FFI calls with automatic GError handling:
1. Takes a C FFI call expression (`nnkCall`)
2. Auto-injects `addr error.error` as the last parameter (GError output)
3. If GError is set → raises `OperationError`
4. If return type is `gboolean` → checks result == 1, raises on failure, returns nothing
5. Otherwise → returns the actual result (raw C pointer)

C functions using `check` **must** have `ptr ptr GError` as their last parameter. The caller does NOT pass it.

### Manual Error Handling
Use when you need finer control (e.g., `validate` returns `bool` while also having a GError):
```nim
var err = newError
let result = some_function(err.toPtr)
if err:
  raise newException(OperationError, "Failed: " & $err)
```

### Exception Boundaries
- Internal step procs should raise (don't catch in the same layer)
- Catch only to recover, translate at a boundary, record failure, or clean up
- Keep the success path straight-line between boundaries
- Translate low-level errors only at module/subsystem boundaries — preserve the original message
- Use specific existing exception types (`ValueError`, `IOError`); add custom types only if callers need distinct handling
- Use `raises` on exported procs when the surface is narrow and stable
- Catch `CatchableError` as the recoverable catch-all, never bare `Exception`
- Use `try/finally` for cleanup, never `try/except`

### Failure Channel Selection
- **Raise** for: invalid input, semantic/validation failure, operational/runtime failure
- **Return `bool`** for: probe-style "not found" with no value payload
- **Return `Option[T]`** for: value type with no natural sentinel, composable via `map`/`flatMap`
- **Return `int`** (parse-length) with `var` out-param for: scanning/parsing helpers
- **Structured result objects** only at: orchestrator/batch boundaries

## Memory Management (ARC/ORC)

### Using `arcGObject` (Preferred for GObject types)
From `utils.nim` — auto-generates `=destroy`, `=sink`, `=copy`, `toPtr` for all types in the block:
```nim
import ../core/[utils, ffi]

arcGObject:
  type
    FileInfo* = object
      handle*: ptr GArrowFileInfo
    FileSelector* = object
      handle*: ptr GArrowFileSelector
```

### Using `arcRef` (Custom ref/unref functions)
```nim
arcRef("g_uri_unref", "g_uri_ref"):
  type
    Uri* = object
      handle*: ptr GUri
```

### Manual Implementation (only for special cases)
- Types with inheritance (`of RootObj`) — macros can't handle
- Types with custom cleanup logic (e.g., `OutputStream` flushes on destroy)
- Types with multiple handles (e.g., `RecordBatchReader` with streamHandle)
- Generic types (macros don't support generics)
- **Keep manual types in separate `type` blocks from macro-wrapped types**

```nim
type
  Array*[T] = object
    handle: ptr GArrowArray

proc `=destroy`*[T](ar: Array[T]) =
  if not isNil(ar.handle):
    g_object_unref(ar.handle)

proc `=sink`*[T](dest: var Array[T], src: Array[T]) =
  if not isNil(dest.handle) and dest.handle != src.handle:
    g_object_unref(dest.handle)
  dest.handle = src.handle

proc `=copy`*[T](dest: var Array[T], src: Array[T]) =
  if dest.handle != src.handle:
    if not isNil(dest.handle):
      g_object_unref(dest.handle)
    dest.handle = src.handle
    if not isNil(dest.handle):
      discard g_object_ref(dest.handle)

proc toPtr*[T](ar: Array[T]): ptr GArrowArray {.inline.} =
  ar.handle
```

### Ownership Hook Rules
- Only write hooks when the type owns a non-managed resource (raw pointers, file descriptors, C handles)
- `=destroy` signature: `x: T` (not `var T`) — prevents accidental field mutation
- `=wasMoved`: reset every field `=destroy` checks (nil pointers), compiler eliminates the subsequent destroy
- `=copy`: in deep-copy types, guard against self-assignment (`if dest.data == src.data: return`)
- `=sink`: don't write custom unless necessary; compiler synthesizes from `=destroy` + raw move
- `=dup`: deep-owning → mark `{.nodestroy.}`, build fresh copy; refcounted → increment counter, share pointer
- Hooks must be defined in the same module as the type; declare all hooks before any proc uses the type
- Safe order: type def → `=destroy` → `=wasMoved` → `=copy` → `=dup`
- Use `sink` parameters and move semantics for zero-copy patterns
- Use `lent T` for immutable borrow accessors into container storage

### Move Semantics
- `move(x)` — source left in moved-from state, `=wasMoved` runs
- `ensureMove(x)` — compile-time only, works for rvalues/sink params, fails for lvalues with destructors
- `sink` params are affine: callee may consume once, or not at all
- When compiler can't prove last-use for a sink arg, it inserts `=copy`/`=dup`

### Ownership Models
| Model | Hooks needed |
|-------|-------------|
| Plain / auto-managed | None |
| Borrowing / view | None + `lent T` accessors |
| Move-only owner | `=destroy`, `=wasMoved`, `=copy` as `{.error.}` |
| Deep-owning container | `=destroy`, `=wasMoved`, `=copy`, `=dup` |
| Shared / refcounted | `=destroy`, `=wasMoved`, `=dup`, `=copy` |

## API Design

- Prefer plain `object` for public data; use `ref object` only when identity/aliasing/shared mutation is required
- Use stdlib names when behavior matches: `len`, `contains`, `[]`, `[]=`, `items`/`mitems`, `pairs`/`mpairs`, `push`/`pop`
- Value types: `initX()` returning `T`; Ref types: `newX()` returning `ref T`; Conversions: `toX()` overloaded
- Accept `openArray` for batch inputs
- Separate required lookup (raises specific exception) from optional lookup (`Option[T]`, `hasKey`, etc.)
- Export only the stable surface; keep helpers private
- Use `distinct`, `Natural`, `Positive`, ranges, enums, sets in type constraints instead of weakening to `int`
- Gate version-specific API with `when (NimMajor, NimMinor) >= (x, y)`, not `{.since.}`
- Don't expose scalar `var` accessors from internal state
- Don't return `lent`/`var` through a temp local — ORC rejects the borrow

### Nim-ified API Conventions
- `len` instead of `get_length`
- `[]` instead of `get_value(i)`
- `$` for string representation (wraps `garrow_*_to_string`)
- Iterators: `items`, `pairs`, `chunks`
- dátummal: `@` for `toSeq` conversion

## Architecture Principles

### Use Battle-Tested Arrow Code
**Never reimplement Arrow's own functionality.** Arrow C++ has hardened years of three-valued logic, NaN handling, type coercion, and statistics pruning. Instead use:
- **Acero execution engine** (`garrow_execute_plan_*`) for in-memory filtering
- **Arrow Dataset GLib** (`gadataset_scanner_builder_set_filter`) for push-down filtering with statistics-based pruning
- **Arrow compute kernels** (`garrow_function_find` + `garrow_function_execute`) for operations

### Layered Implementation
Complex features like filtering: Layer 1 (simple, existing bindings) → Layer 2 (Arrow Dataset integration) → same public API across layers.

###acci  YAGNI & Keep It Simple
- Named parameters over builder patterns unless state accumulation is truly needed
- Fail fast: raise `KeyError` immediately for missing columns
- No parallel reading, batch sizing, or threading knobs until benchmarks prove they help
- No `ReadOptions` objects until multi-file support actually needs them
- Safety first: every `ptr GArrow*` must be wrapped in a Nim `object` with ARC hooks

## Testing

Use `unittest2`:
```nim
import unittest2
import ../src/narrow

suite "Feature Name":
  test "description":
    let arr = newArray(@[1'i32, 2, 3])
    check arr.len == 3
    check arr[0] == 1
    expect(IndexDefect):
      discard arr[-1]
```

Quick test runs: `just test-debug-par` (8-core parallel with ASAN).

**DoAssert vs assert:** In Nim, `assert` is compiled out in `-d:danger`. Use `doAssert` in tests. `AssertionDefect` inherits from `Defect`, not `CatchableError` — bare `except:` won't catch it.

### Test Fixtures (`tests/testfixture.nim`)
Isolated temp directories under `/tmp/narrow/runs/<runid>/<suite>/<test>/` with file locking for thread safety. Use `/` operator: `let path = fixture / "data.parquet"`. Cleanup in teardown.

Note: known upstream Arrow GLib memory leak in CSV `addSchema` (documented in `notes/memory_leak.md`). LSAN suppressions in `lsan.supp`.

## Debugging

### Build Modes
| Mode | Stack frames | Overflow checks |
|------|-------------|-----------------|
| debug (default) | Full file+line | Yes |
| release | Raising frame only | Yes |
| danger | Raising frame only | No |

Restore full traces: `--lineTrace:on` (implies `--stackTrace:on`).

### Techniques
- **Echo inspection**: `echo` (auto-flushes), `repr` for types without `$`, `debugEcho` inside `{.noSideEffect.}` procs. For `stdout.write` (no auto-flush), call `stdout.flushFile()` after.
- **Stack trace**: `writeStackTrace()`
- **Compile-check**: `compiles(expr)` — true if type-checks
- **Macro expansion**: `nim c --expandMacro:<name> file.nim`
- **ARC inspection**: `nim c --expandArc:<proc> file.nim` — shows injected `=copy`/`=destroy`/`=sink`/`move`
- **Memory errors**: AddressSanitizer (primary): needs `-d:useMalloc -d:noSignalHandler --passC="-fsanitize=address ..." --passL="-fsanitize=address ..." -g`. **Both** `--passC` and `--passL` required. With clang, ensure `llvm-symbolizer` in PATH.
- Never use gdb — name mangling makes variable inspection unreliable. Use ASan or echo-based debugging.
- Test under the same `--mm:` mode the project uses.

## Code Organization

- Use explicit state objects for multi-step orchestration: plain `object` passed by `var`. Use `ref object` only when identity/aliasing is required.
- Keep helpers at top-level; nested procs for truly local logic only. Never hide orchestration state in outer locals.
- One module = one piece of state + its operations. Keep exports intentional (narrow surface, not all-of-everything).
- Use `case` on data before reaching for `method`. Model kind in data, not runtime dispatch.

---

## Nim-to-C Binding Patterns

- Two-layer design: raw FFI (`importc` + `cdecl`, thin, ABI-faithful) + ergonomic Nim layer (wraps with `arcGObject`, raises exceptions, Nim-ified names)
- Never reorder C struct fields. Use `packed` only when C specifies it.
- For C enums: `distinct` integer types + `const` values. For bitflags: `distinct` integer + bitwise helpers.
- C types map: `char*` → `cstring`, `void*` → `pointer`, `T*` → `ptr T`, `const char*` → `cstring`. Never pass Nim `string` directly.
- Don't pass Nim closures as C callbacks — use `{.cdecl.}` plain procs + global table keyed by userdata.
- Verify with `static: doAssert sizeof(T) == N` and `offsetOf` checks.

---

## Implementation Checklist for Adding New Types
1. Wrap FFI pointer in an `object`
2. Implement `=destroy`, `=copy`, `=sink` (use `arcGObject` macro where possible)
3. Wrap C calls in `check()` macro for GError handling (or manual for special cases)
4. Export a `handle` or `toPtr` for low-level access
5. Add `unittest2` test case in `tests/`
